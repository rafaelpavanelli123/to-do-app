# To-Do List com Engenharia de Prompt

## Sobre o Projeto

Este projeto consiste no desenvolvimento de uma aplicação de lista de tarefas (To-Do List), com foco na utilização de Inteligência Artificial como ferramenta de apoio durante o processo de desenvolvimento.

O objetivo principal não foi apenas criar a aplicação, mas aplicar conceitos de Engenharia de Prompt, pensamento crítico e compreensão do código gerado.

---

## Objetivo

* Criar uma aplicação funcional de lista de tarefas
* Utilizar IA como suporte no desenvolvimento
* Praticar a construção de prompts estruturados
* Garantir entendimento completo do código gerado

---

## Funcionalidades

* Adicionar novas tarefas
* Definir prioridade (Alta, Média, Baixa)
* Visualizar lista de tarefas dinamicamente
* Marcar tarefas como concluídas
* Remover tarefas
* Estilização condicional baseada na prioridade
* Indicador visual de tarefas concluídas
* Contador de tarefas (total, pendentes e concluídas)

---

## Engenharia de Prompt

Durante o desenvolvimento, foi utilizado um prompt estruturado com os seguintes elementos:

* Persona: Desenvolvedor front-end experiente
* Contexto: Criação de um To-Do List para projeto acadêmico
* Tarefa: Gerar um arquivo HTML completo com funcionalidades interativas
* Formato: Código completo, organizado e comentado

### Exemplo de Prompt Utilizado

```
Aja como um desenvolvedor front-end experiente.

Contexto:
Estou desenvolvendo um projeto acadêmico de um To-Do List simples. O objetivo é criar uma aplicação funcional com HTML, CSS e JavaScript básico, tudo em um único arquivo.

Tarefa:
Gere um arquivo completo index.html que implemente um To-Do List com as seguintes funcionalidades:
- Campo de input para digitar tarefas
- Seletor de prioridade (Alta, Média, Baixa)
- Botão para adicionar tarefa
- Lista de tarefas exibidas na tela
- Cada tarefa deve mostrar o texto e a prioridade
- Permitir marcar tarefas como concluídas
- Permitir remover tarefas
- Aplicar estilos diferentes para cada prioridade (cores diferentes)
- Aplicar estilo visual para tarefas concluídas (ex: riscado)

Requisitos técnicos:
- Todo o código deve estar em um único arquivo (HTML + CSS + JS embutidos)
- Usar apenas JavaScript puro (sem bibliotecas)
- Código deve ser limpo, organizado e comentado
- Estrutura HTML semântica
- CSS simples, mas bem organizado
- Garantir que funcione abrindo direto no navegador

Formato da resposta:
Retorne apenas o código completo do arquivo index.html, sem explicações fora do código.
```

---

## Decisões Técnicas

Apesar do enunciado mencionar HTML e CSS, foi necessário utilizar JavaScript para implementar funcionalidades dinâmicas como:

* Adição de tarefas
* Atualização da lista
* Marcação de conclusão

Sem JavaScript, essas funcionalidades não seriam possíveis apenas com HTML e CSS.

## Reflexão Crítica

Durante o desenvolvimento, foi possível observar que:

* A IA acelera o processo, mas exige validação constante
* A qualidade do resultado depende diretamente da clareza do prompt
* É essencial compreender o código gerado, e não apenas utilizá-lo
* A IA funciona melhor como assistente do que como substituta

Além disso, foi necessário revisar e adaptar as sugestões geradas para garantir aderência aos requisitos do projeto.
